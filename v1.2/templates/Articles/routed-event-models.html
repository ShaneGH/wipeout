Routed Event Models

Routed events are generally for the view model layer. They travel up and down the view model stack, which is essentially the DOM, and do not interact with the model layer. Models can interact with routed events by inheriting from the class

The routed event model is a type known by the view model layer. When a routed event is caught by a view model, it:

  1. checks to see if it is subscribed to it
  2. checks to see if it's model is a routedEventModel, and if so, informs the model of the routed event
Similarily, routed event models can trigger routed events by delegating their view model to trigger it for them
