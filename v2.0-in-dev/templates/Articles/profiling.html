Profiling

Wipeout has a built in rough and ready profiler. To open it, just open a console window and type wipeout.profile.profile(). Try it on this site!
With the profiler you can:

  • Highlight view model html by hovering over it
  • View the hierarchy of view models by honding ctrl+click (does not always work with clickable items in IE)
  • View the rendering, initialization and template compile time of each view model. In order to see these stats, profiling must be turned on before the item is rendered, so before you call ko.applyBindings(...)
  • Debug the view model and model objects themselves, by opening a console window and clicking on a view model header
  • Turn off proiling by calling wipeout.profile.profile(false).
