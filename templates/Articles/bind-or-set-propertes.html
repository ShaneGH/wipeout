Binding or setting properties

Binding properties is the process of binding the value of a property on one model or view model to that of another model or view model. If the "from" property is a ko.observable then all changes to it will reflect in the "to" property. If the "to" property is a ko.observable and the binding is a , then then all changes to it will reflect in the "from" property.

camelCasing

Html editors do not like upper case letters, and although wipeout templates are xml (not html) you will most likely be uing a html editor to create them. So, within wipeout, property names contaianing "-" will convert the property name to camel case on the view model. So, my-property="true" in the template will set myProperty = true on the view model.
