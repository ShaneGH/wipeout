Wipeout

MVVM on steroids for knockout.




The power of knockout


The declaritive nature of XAML


The extensability of an OO framework

Why wipeout?

Wipeout is a template engine for knockout. It leverages the superb client binding and templating capabilities of knockout, but goes one step further, introducing the basic functionalities and development patterns of XAML in a HTML/javascript environment.

What does it look like?

Wipeout is built to look as much like WPF and Silverlight as possible. You define components in javascript, give them html templates and nest them like with XAML. Have a look:

A collapsable component

View model class 
// create a namespace for our application
window.app = {};

// inherit from the wo.view class
app.collapsable = wo.view.extend(function() {
    this._super("app.collapsable");
    
    this.contentTemplate = ko.observable("");
});
                
// a function which will show or hide the collapsable content
app.collapsable.prototype.toggleContent = function() {
    // "colapsableContent" is the id of the div to show/hide
    // in the template for this view model    
    var collapsableHtml = this.templateItems.colapsableContent;
    $(collapsableHtml).slideToggle("fast");
};
View model Template 
<!-- script id="app.collapsable" -->
                    
<div data-bind="click: toggleContent"
    class="panel-heading"
    style="cursor: pointer">
    <h3 class="panel-title"
        data-bind="text: model().header"></h3>
</div>
<div class="panel-body">
    <div id="colapsableContent">
        <!-- Inbuilt wipeout component to display html -->    
        <wo.contentControl template="contentTemplate"
            woInvisible="true">
        </wo.contentControl>
    </div>
</div>

















Root application component

View model class 
// inherit from the wo.view class
app.rootApplication = wo.view.extend(function () {
    // call the base constructor.
    // the constructor for wo.view takes in a template id.
    this._super("app.rootApplication");
});
View model Template 
<!-- script id="app.rootApplication" -->

<div style="width: 300px; margin: 50px"
     class="panel panel-primary">
    <h4 data-bind="text: model().applicationName"></h4>
    <app.collapsable model="$parent.model().collapsableContent">
        <contentTemplate>
            <div data-bind="text: model().content">
            </div>
        </contentTemplate>
    </app.collapsable>
</div>

Application definition

Model and setup 
var rootApp = document.getElementById("wipeoutApp");  
var model = {
    applicationName: "Wipeout demo",
    collapsableContent: {
        header: "Open/close",
        content: "This is content for the component"
    }
};
ko.applyBindings(model, rootApp);
Inline html definiton 
<!-- html definition -->
                    
<div data-bind="wipeout: app.rootApplication" id="wipeoutApp">
</div>
