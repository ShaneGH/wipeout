Wipeout is an MVVM framework built to harness the power and development style of WPF/Silverlight/XAML for a HTML/javascript environment. Wipeout is build on top of knockout to provide a powerful and familiar binding mechanism along with all of the features which knockout provides

Wipeout is comprised of 2 components:

  1. An simple object oriented framework along with some base classes to implement functionality
  2. A knockout template engine which interacts with the wipeout classes

Why is this useful?

Componentisation, componentisation, componentisation. As a precursor to web components, wipeout allows you to create custom html elements and use them within a template. Components have appearence, functionality and an underlying model. How about a quick example:

A collapsable html component

First you create a view model class. This class inherits from the wo.view class (or one of its ancestors). The view model encapsulates all of the view logic of the component, in this case, the ability to show or hide the body of the collapsable

// create a namespace for the application window.helloWipeout = {}; // create a custom view model which inherits from wo.view helloWipeout.collapsable = wo.view.extend(function() { // call wo.view constructor and give the template id for this view model this._super("collapsable_template"); this.open = true; }); // a function which will show or hide the collapsable content helloWipeout.collapsable.prototype.toggle = function() { // "colapsableContent" is the id of the div to show/hide in the template for this view model if(this.open) this.templateItems.colapsableContent.style.display = "none"; else this.templateItems.colapsableContent.style.display = null; this.open = !this.open; };

The collapsable view model needs an appearence, and for this we create a template.

<script type="text/xml" id="collapsable_template"> <!-- a link bound to the toggle function of the view model --> <a href="javascript: void(0)" data-bind="click: toggle">Toggle</a> <!-- a div whos content is bound to the model of the view model. Its id is used to find this element within the view model --> <div data-bind="html: model" id="colapsableContent"></div> </script>

And this is how we use the new view model component

<helloWipeout.collapsable model="'This is the content to be shown/hidden'"></helloWipeout.collapsable>
