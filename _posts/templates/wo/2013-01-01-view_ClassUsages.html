wo.view is the base class for all wipeout view models. The view can be used in 2 ways.

  1. To be inherited from in order to provide visual, model and binding functionalty to view models
  2. As a dummy view model which a template can be attached to

Inheriting from a view

The following creates a simple view model with collapsable content.

Javascript:

// create a custom view model window.collapsable = wo.view.extend(function() { // call wo.view constructor and give the template id for this view model this._super("collapsable_template"); this.open = true; }); // a function which will show or hide the collapsable content window.collapsable.prototype.toggle = function() { // "colapsableContent" is the id of the div to show/hide in the template for this view model if(this.open) this.templateItems.colapsableContent.style.display = "none"; else this.templateItems.colapsableContent.style.display = null; this.open = !this.open; };

Html template:

<script type="text/xml" id="collapsable_template"> <!-- a link bound to the toggle function of the view model --> <a href="javascript: void(0)" data-bind="click: toggle">Toggle</a> <!-- a div whos content is bound to the model of the view model. Its id is used to find this element within the view model --> <div data-bind="html: model" id="colapsableContent"></div> </script>

Usage:

<collapsable model="'This is the content to be shown/hidden'"></collapsable>

Using a view as a dummy view model to render a template

Sometimes you may wish for a model to have an appearance without the need for dynamic view model code. In this case a wo.view can be used directly:

Html template:

<script type="text/xml" id="displayPerson_template"> <label>First name:</label> <span data-bind="html: model().firstName"></span> <label>Last name:</label> <span data-bind="html: model().lastName"></span> </script>

Usage:

<wo.view model="{firstName: 'John', lastName: 'Smith'}" templateId="'displayPerson_template'"></wo.view>
