Self removing items

All items controls subscribe to the removeItem routed event. This means that if a view model in which is a decendent of the itemsControl fires a removeItem routed event with a specic model as the event args, then the model will be removed from the itemSource of the itemsControl.

If this is likely to happen it is important to bind the itemSource of the itemsControl two ways so that changes will propogate back up the model chain.
