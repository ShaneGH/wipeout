Set complex properties

Complex properties with multiple levels can also be set in xml. If there is no parser to go with the constructor property, then the constructor is used as an actual constructor with the new keyword

<wo.contentControl> <!-- outer contentControl --> <mvVal constructor="wo.contentControl" innerValue="'Hello wipeout'"> <!-- inner contentControl --> <template> Binding to element value: <span data-bind="text: innerValue"></span> </template> </mvVal> <template> Rendering view model: <div data-bind="render: mvVal"></div> </template> </wo.contentControl> <wo.view templateId="'set-complex-properties-script-1'" />

This is a fairly complex piece of functionality.

  1. We constructed an outer wo.contenentControl
  2. We constructed an inner wo.contenentControl
  3. We set the innerValue property of the inner contentControl to "Hello Wipeout"
  4. We set the template property of the inner contentControl.
  5. We set the myVal property of the outer contentControl to the innerContentControl
  6. We set the template property of the outer contentControl.
  7. Within the template of the outer content conrol, we use the render binding to render its myVal property.
