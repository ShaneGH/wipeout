Wipeout Native Classes

Wipeout has several native classes which can be inherited from to build an application

This is the base class for all wipeout objects and contains functionality to extend itself and call base methods.

This class derives from wo.object and has the ability to raise routed events (TODO: link) in any view model which it is the model for.

This class derives from wo.object and is a class with a visual element. This should not be inherited from directly, but rather you should inherit from

This class derives from wo.visual and is the base class for view models.

This class derives from wo.view and provides functionality to specify the template as a template string rather than as an Id.

This class derives from wo.contentControl and provides list building functionality. This is the wipeout equivelant of the "foreach" knockout binding

This class derives from wo.contentControl and provides functionality similar to the "if" knockout binding. By default, wo.if has set to true, meaning it will have no binding context.
