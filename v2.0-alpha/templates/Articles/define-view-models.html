Define a view model

View model definitions create view model classes and determine how a view model will act once it is created by wipeout.

The most basic view model definition looks like this:

wo.viewModel("myNamespace.myViewModel").build();

What this definition will do is:

  1. Create a window.myNamespace object. If one exists already, do not overwrite it.
  2. Create a myViewModel function. myViewModel will be the view model class. In order to implement the most basic view model functionality (templates, lifecycle management etc...) the view model will inherit from wo.view by default.
  3. Append the myViewModel to window.myNamespace, (as window.myNamespace.myViewModel).
Always remember to build(...) view models after all definitioins.
