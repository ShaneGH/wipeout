Find a View Model

You can find a view model with the $find api. The find api accepts many search term types which are customisable. For each find type you can either it's full name or its alias.

Search for a view model type

You can select an ancestor view model by passing a view model type into the find block.

<myViewModel model="'Hello wipeout!'"> <template> <wo.content-control my-view-model-ancestor="$context.find(myViewModel)"> <template> <span data-bind="text: myViewModelAncestor.model"></span> </template> </wo.content-control> </template> </myViewModel>

Search for an ancestor

You can select an ancestor by passing a string into the find block. The string must be of the form 'parent', 'grandparent', 'greatgrandparent', 'greatgreatgrandparent', etc...

<my-view-model model="'Hello wipeout!'"> <template> <wo.content-control my-view-model-ancestor="$context.find('parent')"> <template> <span data-bind="text: myViewModelAncestor.model"></span> </template> </wo.content-control> </template> </my-view-model>

Using more complex filters

You can pass a filters object into $call for a targeted search.

<my-view-model model="'Hello wipeout!'"> <template> <wo.content-control my-view-model-ancestor="$context.find({$instanceof: myViewModel})"> <template> <span data-bind="text: myViewModelAncestor.model"></span> </template> </wo.content-control> </template> </my-view-model>
Search type Alias Description
$instanceof $i Find an object which is an instanceof the given search term. Is the same as passing a view model type into $find
$type $t An exact type match. Is the same as currentViewModel.constructor === searchTerm.
$ancestory $a Find the parent, grandparent ect... Is the same as passing a string into the $context.find() method.
$number $n Describes when to stop when a view model is found. For example, if $n === 0, return the first match. If $n === 1, return the second match.
$model $m If set to true, signifies that we are looking for models, not view models.
anything else none Anything on the search object which is not prefixed with a $ will be treated as a property search. For instance using { myProperty: "hello"} will return the first view model where myProperty === hello .
