Call a method of an ancestor

Using knockout event bindings always sets the context of your functions to the current scope. However, when using ancestors function, it is generally more useful to have ancestor obect as the scope. The $call API will do this

// create a custom class to distinguish type window.myType = wo.contentControl.extend(function myType() { this._super(); this.example = "Call the method of an ancestor: "; }); myType.prototype.click = function(text) { alert(this.example + text); }; <myType model="'Hello wipeout!'"> <template> <wo.content-control> <template> <button data-bind="click: $call(myType).dot('click').args('Hello wipeout!')">Test</button> </template> </wo.content-control> </template> </myType> <wo.view templateId="'call-method-out-of-scope-script-1'" /> There is $call, documented .
