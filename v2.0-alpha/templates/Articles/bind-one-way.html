One way bindings

(TODOD diagram) The default binding is one way. This means that the object where the propery is set will recieve updates when the property changes, however, if the object where the property is set changes the value itself, the original owner of the property will not be notified. In most cases properties will bound one way by default, but occasionally (for instance if a default binding is set for a property (TODOD link)) you may need to explicitly set the binding to one way using the --ow flag (TODOD link).
