View Models

Using simple functions as view models is ok for the most basic of functionality, however, in order to access the rich set of wipeout features, you must create view models.

// view model 1 wo.viewModel("app.rootApplication") .templateId("/root-application.html") .build(); // view model 2 wo.viewModel("app.innerViewModel") .templateId("/inner-view-model.html") .build();

At the URL /root-application.html

<h1>Welcome to wipeout</h1> <div> <app.inner-view-model> </app.inner-view-model> </div>

And at the URL /inner-view-model.html

<span>It supports templates</span>

So an app.root-application element in html will render:

<h1>Welcome to wipeout</h1> <div> <span>It supports templates</span> </div>
