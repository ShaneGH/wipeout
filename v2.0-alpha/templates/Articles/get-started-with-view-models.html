viewModels

Using simple functions as viewModels is ok for the most basic of functionality, however, in order to access the rich set of wipeout features, you must create viewModels.

// viewModel 1 wo.viewModel("demoApp.rootApplication") .templateId("/root-application.html") .build(); // viewModel 2 wo.viewModel("demoApp.innerViewModel") .templateId("/inner-view-model.html") .build();

At the URL "/root-application.html"

<h1>Welcome to wipeout</h1> <div> <!-- as you can see, the root-application contains an inner-view-model --> <demo-app.inner-view-model> </demo-app.inner-view-model> </div>

And at the URL "/inner-view-model.html"

<span>It supports templates</span>

So an &lt;app.root-application> element in html will render:

<h1>Welcome to wipeout</h1> <div> <span>It supports templates</span> </div>
