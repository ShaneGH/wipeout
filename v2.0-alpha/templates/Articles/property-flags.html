Property flags

Sometimes you might like to say something about the property within the property itself. Examples of this are:

  1. (TODOD, link) Defining how two properties will be bound (one way, two way etc...)
  2. (TODOD, link) Defining how the string value in the property will be parsed

In order to achieve this you can use property flags. Property flags are a method of splitting a property name into two parts. The first part is the property name, the second is a list of flags.

The seperator for property flags is "--" and each subsequent flag is seperated by a "-".

For instance the model property setting in the view model: &lt;wo.view model--nb-s="Hello wipeout!">&lt;/wo.view> uses the (TODOD link) "No Binding" binding and the (TODOD link)"String" parser.
