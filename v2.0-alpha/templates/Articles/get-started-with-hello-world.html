Hello World

The minimum you need to build a wipeout app is a view model, a html template and some initialization code through knockout

The view model

// create a namespace for our application window.helloWipeout = {}; helloWipeout.rootApplication = wo.view.extend(function rootApplication() { this._super("helloWipeout.rootApplication"); });

Here, we are creating a view model class. The view model has to inherit from or one of its derived classes. The call to _super(...) calls the constructor of the parent class, in this case wo.view. The constructor for a wo.view takes a template id as an argument

The Template

<script type="text/xml" id="helloWipeout.rootApplication"> <div> Hello <span data-bind="text: model().name"></span>! </div> </script>

The context or scope of the template is always going to be the view model, in this case a helloWipeout.rootApplication object. Because helloWipeout.rootApplication inherits from , it has a model proerty which is a ko.observable. We are going to bind to the name property on the model.

Creating the application

<div id="helloWipeoutApp" data-bind="wipeout: helloWipeout.rootApplication"></div> <script type="text/javascript"> var rootApp = document.getElementById("helloWipeoutApp"); ko.applyBindings({ name: "Wipeout" }, rootApp); </script>

Within the html document, we now specify a placeholder for our application and tell it which view model type to use using the binding.
Finally, we apply all knockout to the placeholder div and give it the root model.

This will ultimately render: <div> Hello <span>Wipeout</span>! </div>

When you apply the wipeout binding, it will search for the view model "helloWipeout.rootApplication". An instance of this view model is created and it's model property will be set with the value passed into ko.applyBindings. This is where the "Wipeout" in "Hello Wipeout!" comes from.

So, all together:
