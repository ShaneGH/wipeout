Two way bindings

By default, all bindings are one way. This means that the object where the propery is set will recieve updates when the property changes, however, if the object where the property is set changes the value itself, the original owner of the property will not be notified. This can be remedied by using a two way binding. Two way bindings are suffixed with -tw.

<wo.content-control model="{ value: ko.observable('Hello wipeout!') }"> <template> <wo.content-control model-tw="$parent.model().value"> <template> Two way bindings: <span data-bind="text: model"></span> </template> </wo.content-control> </template> </wo.content-control> <wo.view templateId="'bind-two-way-script-1'" />

Now if the child content control modifies it's model, the resulting change will reflect in the parent's model.value. Notice how the value is an observable in this case. Two way bindings can only be defined between two observable properties
