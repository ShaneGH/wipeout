List item lifecycle

You can inherit from the itemsControl to create lists with tighter control on the view models they render.

With a custom itemsControl you can control and .

var myItemsControl = wo.viewModel("myItemsControl", wo.itemsControl) .build(); myItemsControl.onItemRendered = function(item) { this._super(item); // do initialization logic here }; myItemsControl.onItemRemoved = function(item) { this._super(item); // do cleanup logic here }; Usage: <my-items-control items="...">...</my-items-control>

Remember that onItemRendered and onItemRemoved are methods which already exist on the wo.itemsControl class. When you override a method of a parent class it is generally necessary to call the method so that the method of the parent class is still invoked.
